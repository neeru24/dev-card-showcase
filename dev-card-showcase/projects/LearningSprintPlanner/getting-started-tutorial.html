Step-by-Step Tutorial

1

Explore the Dashboard

First, explore the dashboard. You'll see your current streak, study hours, and active goals all in one place. This is your command center for all learning activities. Get familiar with the interface and available sections.

Step 1
2

Create Your First Goal

Navigate to "Study Goals" tab and click "Add New Goal". Set a realistic, measurable goal (e.g., "Complete JavaScript Basics in 2 weeks"). Goals keep you motivated and on track. Make them SMART: Specific, Measurable, Achievable, Relevant, Time-bound.

Step 2
3

Organize Your Topics

Go to "Topics" section and add subjects you want to study. You can categorize them by difficulty level or priority. Break down complex subjects into smaller, manageable topics for better focus and faster progress.

Step 3
4

Start Your First Session

Click "Study Sessions" and select a topic. Choose your preferred session duration (25, 45, or 90 minutes). Use the built-in timer to maintain focus and block distractions. Consistency is key to building learning habits.

Step 4
5

Monitor Your Progress

Visit the "Progress" tab to see visual analytics. Track topics completed, study hours logged, and maintain your daily streak. This data motivates continuous improvement and helps you identify patterns in your learning.

Step 5
6

Test Your Knowledge

Take quizzes in "Quiz Analytics" section to assess your learning. Get instant feedback and see where you need improvement. Repeat questions to strengthen weak areas and reinforce understanding.

Common Mistakes to Avoid

Vague Goals
Starting with unspecific goals like "Get better at coding"
Set SMART goals: "Complete 10 JS algorithms in 1 week"
Random Topic Selection
Adding topics without prioritization or logical sequence
Organize from basics to advanced, respect prerequisites
Ignoring Analytics
Not reviewing progress charts or quiz results regularly
Check progress weekly and adjust your plan
Untested Code PRs
Submitting code changes without verifying functionality
Test thoroughly in browser DevTools before PR
Ignoring Guidelines
Not reading code style or commit message conventions
Always check CONTRIBUTING.md before submitting
Inconsistent Learning
Studying sporadically without maintaining a regular schedule
Set daily study time and build lasting habits

Troubleshooting & FAQ

Data not saving to localStorage?

Issue: Your goals and study sessions disappear after refresh.

Solution: Check if localStorage is enabled in your browser. Some browsers block storage in private/incognito mode. Clear browser cache and try again. Make sure JavaScript is enabled. Check console for errors.

Timer not working correctly?

Issue: Timer counts incorrectly or doesn't stop at zero.

Solution: Check browser console for JavaScript errors. Try closing and reopening the page. Ensure your system time is correct. Try a different browser to isolate the issue. Clear cache if necessary.

Progress charts not showing?

Issue: Analytics page is blank or charts don't display.

Solution: Charts require at least one study session to display data. Add a topic and complete a study session first. Check if JavaScript charting library is loaded. Refresh the page with Ctrl+Shift+R to do a hard refresh.

How do I create a pull request?

Steps:
1. Fork the repository on GitHub
2. Create a feature branch: git checkout -b feature/name
3. Make changes and commit: git commit -m "Add: description"
4. Push to your fork: git push origin feature/name
5. Go to GitHub and click "New Pull Request"
6. Write a clear description and submit for review

I messed up my commits. What do I do?

Solution before pushing:
git reset --soft HEAD~1 - Undo last commit, keep changes
git reset --hard HEAD~1 - Undo last commit, discard changes

After pushing: Contact maintainers or submit a corrected PR. No worries!
