Help & Documentation

Overview

The Decoupled Event Propagation Backbone provides a centralized event system for managing communication between loosely coupled components in event-driven architectures.

Key Concepts

  • Components: Independent modules that publish or subscribe to events
  • Events: Messages that carry data between components
  • Event Bus: The central mediator for event propagation
  • Subscriptions: Rules defining which components receive which events

How to Use

  1. Add components to the architecture view
  2. Configure event subscriptions for each component
  3. Publish events to see propagation in action
  4. Monitor performance and event flow

Propagation Modes

Synchronous
Events are processed immediately by all subscribers
Asynchronous
Events are queued and processed in the background
Batched
Multiple events are grouped and processed together

Best Practices

  • Use descriptive event names
  • Keep event data lightweight
  • Handle errors gracefully
  • Monitor system performance
